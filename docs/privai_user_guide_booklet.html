Your private local AI workspace

For first-time users

Follow the guide from the starting page to your first conversation. Each section explains what you see and what to do next.

Privacy-first by design

Chat content, uploaded files, converted files, and model inference stay on the device. Firebase is used only for identity and device heartbeat metadata.

1. Start at the home screen

What you see

  • The Privai logo in the top-left corner.
  • A blue New chat button for creating a fresh conversation.
  • The conversation list, which is empty on first use.
  • Navigation links for Device, How it works, and Settings.
  • The account panel at the bottom with theme and sign-out controls.
  • A model status pill in the top-right showing whether Ollama is idle or loaded.
  • The composer at the bottom, where you type messages and choose modes.
Before the first real chat, connect the web app to your backend device in Settings. The web app is only the interface; the model, database, attachments, and conversion tools live on the device.

2. Connect your board in Settings

Step-by-step

  1. Open Settings from the sidebar.
  2. In Device URL, enter the backend URL. During local development this is usually http://127.0.0.1:8080.
  3. Click Save. If the backend is online, the page shows reachable and the active model name.
  4. Pair your account using the code printed by the backend on first boot. After pairing, only your Firebase account can talk to this device.

Why pairing matters

Privai is designed so the browser talks directly to the device. Firebase handles sign-in, but the device still verifies that the request belongs to the paired owner. That prevents another signed-in account from using your local model or reading your local chat database.

The privacy posture box is important: chat content is not sent to Firebase. Conversations live in SQLite on the device, and uploaded or converted files live in local attachment storage.

3. Choose a model on the Device page

LLM lifecycle

The LLM lifecycle card controls whether the selected model is loaded in memory. When the model is loaded, Privai can answer faster but uses RAM. When stopped, RAM is freed and the next response may take longer.

Model picker

The model picker reads installed Ollama models from the device. Choose a model based on your hardware and the task:

Model type Best for Tradeoff
Small text model Fast chat, summaries, simple reasoning Weaker at long HTML artifacts and complex instructions
Larger text model Better writing, code, and website artifacts Uses more RAM and runs slower
Vision model Understanding attached images Must be installed separately, for example with Ollama
If a model is not listed, install it on the device first, for example with ollama pull qwen2.5:7b, then click Refresh.

5. Start a new chat

The four controls in the composer

Web

Adds current context from local SearXNG search. Use it for news, recent facts, product versions, citations, or anything that might have changed.

Agent

Build mode. Use it when you want a website, calculator, game, chart, HTML artifact, or code-like output. Agent mode follows local build guidelines.

Convert

Local file conversion. Attach a file, choose convert, then type something like to pdf or to heic.

Attachments

Add images, PDFs, text files, or documents. Attachments stay on the device and can be used for chat, vision, document context, or conversion.

Core workflows

Workflow A - Ask a normal private question

  1. Click New chat.
  2. Leave Web, Agent, and Convert off.
  3. Type your question.
  4. Press Enter or click Send.

Use this for private brainstorming, summaries, drafting, translations, and local notes. The model answers from its local knowledge.

Workflow B - Research with sources

  1. Click Web.
  2. Ask for sources, recent information, or references.
  3. Privai searches through local SearXNG, filters results, and passes a small number of relevant snippets to the model.

Web search is useful, but it is the least private mode because search queries can be sent to upstream engines. Use it only when freshness or citations matter.

Workflow C - Build a website or artifact

  1. Click Agent.
  2. Describe exactly what to build.
  3. If you need sources, also click Web or ask for sources.
  4. If you have a PDF or image, attach it before sending.

Agent mode uses a local instruction file that tells the model to return a complete HTML artifact, include sources inside the page, and keep the layout simple enough to finish.

Workflow D - Convert files locally

  1. Attach a file.
  2. Click Convert.
  3. Type the target format, for example to pdf or to heic.
  4. Download the converted attachment from the assistant response.

This is safer than public converter websites for private documents, contracts, invoices, personal photos, and school files.

Privacy quick reference

Data Where it goes What to remember
Login Firebase Auth Used for identity and signed ID tokens.
Device heartbeat Firestore Stores status metadata, not chat content.
Chats Device SQLite Conversation text stays on your device.
Attachments Device attachment storage Uploaded and converted files stay local.
Model inference Ollama on device Answers are generated locally.
Web search SearXNG and upstream engines Only use when current web context is needed.
The most important rule: if Web is off, normal chat, attachments, conversion, and local model inference are designed to stay on your own hardware.

Troubleshooting

The app says the backend is unreachable

Open Settings and verify the Device URL. For local development it is usually http://127.0.0.1:8080. Also make sure the FastAPI backend is running.

The model is not listed

Install it on the device with Ollama, then refresh the Device page. Example: ollama pull qwen2.5:7b.

Image questions do not work

Install a vision model and configure OLLAMA_VISION_MODEL. Example: ollama pull llava:7b.

PowerPoint to PDF does not work

Install LibreOffice or soffice on the device. Image conversion uses local image tools; Office conversion needs LibreOffice.

The agent builds weak websites

Use Agent mode and give a concrete request: topic, audience, style, sections, and whether sources should be included. Keep Web sources limited; too many snippets reduce the space available for HTML.

The context meter is full

Use /compact to summarize old turns into one recap, or /clear to wipe the conversation.
