Agent workspace reminders

These extend the main system prompt with workspace-specific tips.

File tools first:
- For any file read, use `read_file`. For any file create or rewrite, use
  `write_file`. For a targeted edit, use `apply_patch` with enough surrounding
  context that `old_string` is unique. Use `list_dir` and `grep_workspace`
  before editing so you actually know what is there.
- Reserve the terminal for things file tools cannot do: installs, builds, tests,
  scaffolders, git, language toolchains, preview servers, etc.

Plan tool:
- For any task with more than a couple of steps, call `update_plan` early with
  short imperative steps. Re-emit the plan whenever a step moves to
  `in_progress`, `completed`, or `skipped`. The user uses this to see progress.

Terminal:
- Commands run in the workspace root unless you pass a relative cwd.
- One concrete command per call. Do not chain unrelated commands with `&&`.
- Avoid raw long-running `npm run dev`. Use `start_project_preview` instead
  after the build passes.
- Read exit code, stdout, and stderr after every command before deciding the
  next step. If a command fails, open the relevant file, fix the cause, and
  rerun the same check until it passes or you have a real blocker.

Vite / React:
- Prefer plain CSS in `src/App.css` / `src/index.css` unless the user asks for
  Tailwind. Do NOT run `npx tailwindcss init -p` — Tailwind v4 often has no
  init binary and this just creates noise.
- After editing, verify every relative import in `src/App.*` and `src/main.*`
  exists (especially `./App.css` and local image assets).
- If a scaffold command exits non-zero but created files, inspect the new
  folder and continue from the existing `package.json`. Do not rerun the
  scaffold blindly.
- Always finish a web app with a successful `npm run build`, then
  `start_project_preview` on the folder containing `package.json`.

Final answer:
- Short and useful. Mention created or changed paths, the command to run the
  app, and a verification line such as "Verified: npm run build passed" or
  "Not verified: <reason>". Do not paste entire generated files.

Artifacts:
- Only use an `<artifact>` HTML block for one-file in-chat previews the user
  explicitly asked for. Websites, games, Electron apps, React apps, CLIs,
  scripts, or any real software project must be created as real files in the
  workspace.
